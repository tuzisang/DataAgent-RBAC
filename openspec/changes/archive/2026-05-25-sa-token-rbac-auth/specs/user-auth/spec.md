## 新增需求

### 需求：用户使用用户名和密码登录
系统必须提供登录接口，通过用户名和密码认证用户，成功后返回 Sa-Token 会话 Token。

#### 场景：登录成功
- **当** 用户向 `/api/auth/login` 提交有效的用户名和密码
- **那么** 系统校验凭据并返回包含用户信息、角色和权限的 Token

#### 场景：登录失败——密码错误
- **当** 用户向 `/api/auth/login` 提交正确的用户名但错误的密码
- **那么** 系统返回 HTTP 401，消息为"用户名或密码错误"

#### 场景：登录失败——账号被禁用
- **当** 用户提交已被禁用账号的凭据
- **那么** 系统返回 HTTP 403，消息为"账号已被禁用"

### 需求：用户登出
系统必须提供登出接口，终止当前 Sa-Token 会话。

#### 场景：登出成功
- **当** 已认证用户调用 `/api/auth/logout`
- **那么** 系统使 Token 失效并返回成功

### 需求：用户注册
系统必须允许新用户使用用户名和密码注册。注册后的默认角色必须是 `viewer`。

#### 场景：注册成功
- **当** 用户向 `/api/auth/register` 提交用户名和密码
- **那么** 系统创建用户（密码哈希存储）并分配 `viewer` 角色

#### 场景：注册失败——用户名已存在
- **当** 用户提交一个已存在的用户名
- **那么** 系统返回 HTTP 400，消息为"用户名已存在"

### 需求：获取当前用户信息
系统必须提供接口供已认证用户获取自己的资料，包括角色和权限。

#### 场景：已认证用户获取信息
- **当** 已认证用户调用 `/api/auth/info`
- **那么** 系统返回用户资料、角色列表和权限列表

#### 场景：未认证用户获取信息
- **当** 未认证用户调用 `/api/auth/info`
- **那么** 系统返回 HTTP 401

### 需求：密码存储安全
系统必须使用 BCrypt 加盐哈希存储用户密码。明文密码不得持久化。

#### 场景：注册时密码存储
- **当** 用户使用密码 "plaintext123" 注册
- **那么** 存储的密码是 BCrypt 哈希值，不是原始字符串
